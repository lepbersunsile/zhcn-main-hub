# zhcn-main-hub

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面的集中存放与展示空间。

## 项目简介

zhcn-main-hub 旨在提供一个结构化的页面归档环境，便于管理、维护和分发独立的 HTML 资源。这些页面可能用于信息展示、工具页面、存档记录等场景，彼此之间无直接关联。

仓库本身不包含动态服务端逻辑，所有内容均为纯前端 HTML 文件，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

```
├── pages/          # 存放所有独立 HTML 页面的主目录
│   ├── example/    # 示例页面（可删除或参考）
│   └── ...
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选，用于导航或说明）
└── README.md       # 本文件
```

- `pages/`：每个子文件夹代表一个独立页面项目，内部包含对应的 HTML 文件及相关资源。
- `assets/`：存放跨页面共享的样式、脚本或媒体文件，避免重复。
- `index.html`：可自定义为入口页面，提供页面索引或简要说明。

## 页面归档说明

- 每个独立页面应存放在 `pages/` 下的单独子文件夹中，命名建议简洁、无歧义。
- 页面间应保持独立，避免相互依赖（共享资源除外）。
- 归档内容可以是已完成、维护中或历史存档的页面，不要求持续更新。
- 本仓库不承诺对任何页面内容的准确性、时效性或可用性负责。

## 维护说明

- 贡献者可通过 Pull Request 添加新页面或更新现有页面。
- 添加新页面时，请在 `pages/` 下创建对应子文件夹，并确保页面能独立运行。
- 如使用共享资源，请将文件放入 `assets/` 目录，并在页面中引用正确路径。
- 请勿在仓库中提交敏感信息（如密码、密钥、隐私数据等）。
- 仓库维护者保留对内容进行整理、移动或删除的权利。

## 使用方式

1. 克隆或下载本仓库。
2. 在 `pages/` 下添加你的 HTML 页面及其资源。
3. 通过本地浏览器打开 HTML 文件，或部署至静态托管服务。
4. 如需通过 GitHub Pages 访问，请确保仓库设置中已启用 Pages 功能。
